4.2 - Levantamento de Requisitos

SIGETRAN - Sistema de Gestão de Transporte

Projeto acadêmico - Disciplina: Projetos de Software II | Prof. Marcos Lopes | Uniube

1. Requisitos Funcionais

IDDescriçãoPrioridadePerfil(s)
RF01 O sistema deve permitir cadastrar, editar, excluir e consultar veículos da frota (placa, modelo, marca, ano de fabricação, capacidade de carga, status ativo/inativo). ALTA Administrador, Gestor de Transporte
RF02 O sistema deve permitir cadastrar, editar, excluir e consultar motoristas (nome, CPF, CNH, telefone, e-mail, status ativo/inativo). ALTA Administrador, Gestor de Transporte
RF03 O sistema deve permitir cadastrar, editar, excluir e consultar clientes (nome, CPF/CNPJ, telefone, e-mail, endereço, status ativo/inativo). ALTA Administrador, Gestor de Transporte
RF04 O sistema deve permitir cadastrar rotas com origem, destino, distância em km, tempo estimado e valor de pedágio. ALTA Administrador, Gestor de Transporte
RF05 O sistema deve permitir cadastrar cargas com descrição, peso em kg, volume em m³ e tipo de mercadoria (Geral, Perigosa, Perecível, Frágil, Granel). ALTA Administrador, Gestor de Transporte
RF06 O sistema deve calcular automaticamente o frete com base na distância da rota, peso da carga, capacidade do veículo, valor por km e pedágios. Adicionais: Perigosa +30%, Perecível +20%, Frágil +15%, Granel +5%. ALTA Financeiro, Administrador
RF07 O sistema deve permitir associar um veículo e um motorista disponíveis a uma viagem programada. ALTA Gestor de Transporte
RF08 O sistema deve registrar viagens com os dados: veículo, motorista, rota, carga, cliente e data de partida prevista. ALTA Gestor de Transporte
RF09 O sistema deve permitir alterar o status da viagem para: Programada, Em Andamento, Concluída ou Cancelada, registrando automaticamente as datas de partida e chegada. ALTA Gestor de Transporte, Motorista
RF10 O sistema deve gerar relatórios de viagens realizadas e fretes calculados, com filtros por período, motorista, veículo e status. MÉDIA Administrador, Financeiro
RF11 O sistema deve permitir consultar o histórico de viagens por veículo e por motorista. MÉDIA Administrador, Gestor de Transporte, Financeiro
RF12 O sistema deve permitir cadastrar usuários com níveis de acesso diferenciados (Administrador, Gestor de Transporte, Motorista, Financeiro). ALTA Administrador
RF13 O sistema deve realizar autenticação por login e senha, gerando token JWT para as sessões. ALTA Todos
RF14 O sistema deve exibir um painel gerencial (dashboard) com indicadores resumidos: viagens ativas, veículos em rota, fretes do mês, alertas. MÉDIA Administrador, Gestor de Transporte
RF15 O sistema deve validar a disponibilidade de veículo e motorista antes de confirmar uma nova viagem. ALTA Gestor de Transporte
RF16 O sistema mobile deve exibir ao motorista apenas suas viagens e permitir atualizar o status. MÉDIA Motorista

2. Requisitos Não Funcionais

IDCategoriaDescriçãoPrioridade
RNF01 Disponibilidade O sistema deve estar disponível 95% do tempo em ambiente de nuvem. ALTA
RNF02 Segurança Todas as requisições à API (exceto login) devem ser autenticadas via token JWT. ALTA
RNF03 Segurança As senhas dos usuários devem ser armazenadas com hash bcrypt. ALTA
RNF04 Usabilidade A interface web deve ser responsiva e funcionar adequadamente em resoluções a partir de 1024x768. MÉDIA
RNF05 Usabilidade O aplicativo mobile deve funcionar em dispositivos Android (versão 8+) e iOS (versão 13+). MÉDIA
RNF06 Desempenho As requisições à API REST devem responder em até 2 segundos em condições normais. MÉDIA
RNF07 Manutenibilidade O código backend deve possuir cobertura de testes (TDD) com Jest/Supertest nas principais rotas. ALTA
RNF08 Portabilidade O backend deve ser containerizável (Docker) para facilitar implantação em diferentes ambientes de nuvem. MÉDIA
RNF09 Compatibilidade A API REST deve seguir os padrões HTTP/JSON, sendo consumível por qualquer cliente (web, mobile, outros). ALTA
RNF10 Rastreabilidade O código deve ser versionado com Git, com histórico de commits descritivos ao longo do desenvolvimento. ALTA
RNF11 Escalabilidade A arquitetura deve permitir a substituição do banco SQLite por PostgreSQL sem refatorações significativas. BAIXA
RNF12 Acessibilidade As interfaces devem seguir boas práticas de acessibilidade (contraste de cores, tamanho de fonte adequado, labels em formulários). BAIXA

3. Regras de Negócio

IDRegra
RN01Um veículo não pode ser alocado a duas viagens simultâneas com status "Programada" ou "Em Andamento".
RN02Um motorista não pode ser alocado a duas viagens simultâneas com status "Programada" ou "Em Andamento".
RN03O peso da carga não pode exceder a capacidade de carga do veículo alocado.
RN04O valor do frete = (distância × R$2,50/km × fator_peso) + pedágio + adicional_tipo_carga. Fator_peso = peso_carga / capacidade_veículo.
RN05A exclusão de um motorista ou veículo não é permitida se houver viagens ativas (Programada ou Em Andamento) vinculadas.
RN06Somente o Administrador pode cadastrar, editar ou excluir usuários do sistema.
RN07Um frete é gerado automaticamente ao criar uma viagem, podendo ser recalculado pelo perfil Financeiro.
