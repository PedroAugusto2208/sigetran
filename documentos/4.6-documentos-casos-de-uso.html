SIGETRAN - Sistema de Gestao de Transporte

Projeto academico - Disciplina: Projeto de Software II

UC01 - Realizar Login
DescricaoPermite que o usuario acesse o sistema mediante autenticacao por login e senha.
AtoresAdministrador, Gestor de Transporte, Motorista, Financeiro
Pre-condicoesO usuario deve possuir cadastro ativo no sistema.
Pos-condicoesO usuario e autenticado e redirecionado ao dashboard conforme seu perfil de acesso.
Fluxo Principal
  1. O usuario acessa a tela de login do sistema.
  2. O usuario informa login e senha.
  3. O sistema valida as credenciais.
  4. O sistema identifica o perfil do usuario (Administrador, Gestor, Motorista ou Financeiro).
  5. O sistema redireciona o usuario para o dashboard correspondente.
Fluxo Alternativo
  1. FA01 - Credenciais invalidas: No passo 3, se as credenciais estiverem incorretas, o sistema exibe mensagem de erro e solicita nova tentativa.
  2. FA02 - Usuario inativo: No passo 3, se o usuario estiver inativo, o sistema exibe mensagem informando que o acesso esta bloqueado.
Requisitos RelacionadosRF13 - Permitir autenticacao por login e senha.
UC02 - Gerenciar Frota (Veiculos)
DescricaoPermite cadastrar, editar, excluir e consultar veiculos da frota.
AtoresAdministrador, Gestor de Transporte
Pre-condicoesO usuario deve estar autenticado com perfil de Administrador ou Gestor de Transporte.
Pos-condicoesO veiculo e cadastrado/atualizado/removido no sistema.
Fluxo Principal (Cadastrar)
  1. O usuario acessa o menu "Frota / Veiculos".
  2. O usuario clica em "Novo Veiculo".
  3. O sistema exibe o formulario de cadastro (placa, modelo, marca, ano, capacidade de carga).
  4. O usuario preenche os dados e clica em "Salvar".
  5. O sistema valida os dados e salva o veiculo.
  6. O sistema exibe mensagem de sucesso e retorna a listagem.
Fluxo Alternativo
  1. FA01 - Editar: No passo 1, o usuario clica em "Editar" em um veiculo existente. O sistema carrega os dados no formulario para edicao.
  2. FA02 - Excluir: No passo 1, o usuario clica em "Excluir". O sistema solicita confirmacao e, se confirmado, remove o veiculo.
  3. FA03 - Placa duplicada: No passo 5, se a placa ja existir, o sistema exibe mensagem de erro.
Requisitos RelacionadosRF01 - Permitir cadastrar, editar, excluir e consultar veiculos da frota.

UC06 - Gerenciar Cargas
DescricaoPermite cadastrar cargas com descricao, peso, volume e tipo de mercadoria.
AtoresAdministrador, Gestor de Transporte
Pre-condicoesO usuario deve estar autenticado com perfil adequado.
Pos-condicoesA carga e cadastrada/atualizada/removida no sistema.
Fluxo Principal (Cadastrar)
  1. O usuario acessa o menu "Cargas".
  2. O usuario clica em "Nova Carga".
  3. O sistema exibe o formulario (descricao, peso em kg, volume em m3, tipo de mercadoria).
  4. O usuario seleciona o tipo de mercadoria (Geral, Perigosa, Perecivel, Fragil, Granel).
  5. O usuario clica em "Salvar".
  6. O sistema valida e salva a carga.
Fluxo Alternativo
  1. FA01 - Editar: O usuario clica em "Editar" para alterar dados de uma carga existente.
  2. FA02 - Excluir: O sistema verifica se a carga nao esta vinculada a viagens antes de permitir exclusao.
  3. FA03 - Peso excede capacidade: Ao vincular a uma viagem, o sistema alerta se o peso da carga excede a capacidade do veiculo.
Requisitos RelacionadosRF05 - Permitir cadastrar cargas com descricao, peso, volume e tipo de mercadoria.
UC07 - Calcular Frete
DescricaoCalcula o valor do frete com base na distancia, tipo de carga, peso, capacidade do veiculo e pedagios.
AtoresFinanceiro, Administrador
Pre-condicoesDevem existir rota, veiculo e carga cadastrados no sistema.
Pos-condicoesO valor do frete e calculado e armazenado, vinculado a viagem correspondente.
Fluxo Principal
  1. O usuario acessa o menu "Calculo de Frete".
  2. O usuario seleciona a rota, o veiculo, a carga e o cliente.
  3. O usuario clica em "Calcular Frete".
  4. O sistema calcula o valor base (distancia x valor por km).
  5. O sistema aplica o fator de peso (peso da carga / capacidade do veiculo).
  6. O sistema soma o valor dos pedagios da rota.
  7. O sistema aplica o adicional por tipo de mercadoria (Perigosa: 30%, Perecivel: 20%, Fragil: 15%, Granel: 5%).
  8. O sistema exibe o detalhamento e o valor total do frete.
Fluxo Alternativo
  1. FA01 - Dados incompletos: Se o usuario nao selecionar todos os campos obrigatorios, o sistema exibe mensagem solicitando o preenchimento.
Requisitos RelacionadosRF06 - Calcular frete com base na distancia, no peso da carga, no valor por quilometro e nos pedagios.
UC08 - Programar Viagem
DescricaoPermite associar um veiculo e um motorista a uma viagem, definindo rota, carga e data de partida.
AtoresGestor de Transporte
Pre-condicoesDevem existir veiculo, motorista, rota, carga e cliente cadastrados. O veiculo e motorista devem estar disponiveis.
Pos-condicoesA viagem e registrada com status "Programada".
Fluxo Principal
  1. O usuario acessa o menu "Viagens".
  2. O usuario clica em "Programar Viagem".
  3. O sistema exibe o formulario (veiculo, motorista, rota, carga, cliente, data de partida).
  4. O usuario preenche os dados e clica em "Programar".
  5. O sistema valida a disponibilidade do veiculo e motorista.
  6. O sistema registra a viagem com status "Programada".
  7. O sistema exibe confirmacao e retorna a listagem.
Fluxo Alternativo
  1. FA01 - Veiculo indisponivel: O sistema informa que o veiculo ja esta alocado em outra viagem no periodo.
  2. FA02 - Motorista indisponivel: O sistema informa que o motorista ja esta alocado em outra viagem no periodo.
Requisitos RelacionadosRF07 - Associar veiculo e motorista a uma viagem. RF08 - Registrar viagens programadas.
UC09 - Atualizar Status da Viagem
DescricaoPermite alterar o status da viagem entre Programada, Em Andamento, Concluida ou Cancelada.
AtoresGestor de Transporte, Motorista
Pre-condicoesA viagem deve existir no sistema.
Pos-condicoesO status da viagem e atualizado. Datas de partida/chegada sao registradas automaticamente.
Fluxo Principal
  1. O usuario acessa a listagem de viagens.
  2. O usuario clica em "Atualizar" na viagem desejada.
  3. O sistema exibe as opcoes de status disponiveis.
  4. O usuario seleciona o novo status.
  5. Se "Em Andamento", o sistema registra a data/hora de partida.
  6. Se "Concluida", o sistema registra a data/hora de chegada.
  7. O sistema salva a atualizacao.
Fluxo Alternativo
  1. FA01 - Cancelar viagem: O usuario seleciona "Cancelada". O sistema solicita motivo do cancelamento.
Requisitos RelacionadosRF09 - Alterar o status da viagem para programada, em andamento, concluida ou cancelada.
UC10 - Consultar Historico de Viagens
DescricaoPermite consultar o historico de viagens por veiculo e por motorista.
AtoresAdministrador, Gestor de Transporte, Financeiro
Pre-condicoesO usuario deve estar autenticado. Devem existir viagens registradas.
Pos-condicoesO historico e exibido conforme os filtros aplicados.
Fluxo Principal
  1. O usuario acessa o menu "Relatorios" ou a listagem de viagens.
  2. O usuario aplica filtros (periodo, motorista, veiculo, status).
  3. O sistema busca e exibe as viagens correspondentes.
  4. O usuario pode visualizar detalhes de cada viagem.
Fluxo Alternativo
  1. FA01 - Sem resultados: O sistema exibe mensagem informando que nenhuma viagem foi encontrada para os filtros selecionados.
Requisitos RelacionadosRF11 - Consultar historico de viagens por veiculo e por motorista.
UC11 - Emitir Relatorios Gerenciais
DescricaoGera relatorios gerenciais de viagens realizadas e fretes calculados.
AtoresAdministrador, Financeiro
Pre-condicoesO usuario deve estar autenticado. Devem existir dados de viagens e fretes no sistema.
Pos-condicoesO relatorio e gerado e disponibilizado para visualizacao/exportacao.
Fluxo Principal
  1. O usuario acessa o menu "Relatorios".
  2. O usuario seleciona o tipo de relatorio (Viagens, Fretes, Utilizacao da Frota, Desempenho Motoristas).
  3. O usuario define o periodo desejado.
  4. O sistema gera o relatorio com os dados consolidados.
  5. O sistema exibe o relatorio em formato tabular.
  6. O usuario pode exportar o relatorio.
Fluxo Alternativo
  1. FA01 - Sem dados: O sistema informa que nao ha dados para o periodo selecionado.
Requisitos RelacionadosRF10 - Gerar relatorios de viagens realizadas e fretes calculados. RF14 - Exibir painel gerencial com indicadores resumidos.
