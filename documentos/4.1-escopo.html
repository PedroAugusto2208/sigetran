4.1 - Definição do Escopo do Projeto

SIGETRAN - Sistema de Gestão de Transporte

Projeto acadêmico - Disciplina: Projetos de Software II | Prof. Marcos Lopes | Uniube

1. Visão Geral do Projeto

O SIGETRAN (Sistema de Gestão de Transporte) é um sistema de informação destinado ao controle e gerenciamento de operações de transporte de cargas. O sistema permite o cadastro e controle de frota de veículos, motoristas, clientes, rotas e viagens, além do cálculo automatizado de fretes.

O sistema será disponibilizado em duas interfaces: web (para uso em computadores por gestores e financeiro) e mobile (para uso por motoristas em campo), ambas acessíveis remotamente via nuvem.

2. Objetivos do Projeto

  • Automatizar o controle da frota de veículos e alocação de motoristas;
  • Gerenciar o ciclo completo de viagens: programação, acompanhamento e conclusão;
  • Calcular automaticamente o valor de fretes com base em regras de negócio definidas;
  • Fornecer relatórios gerenciais para apoio à tomada de decisão;
  • Controlar o acesso ao sistema por meio de perfis de usuário diferenciados.

3. Escopo Incluso

  • Cadastro e gestão de veículos (frota);
  • Cadastro e gestão de motoristas (CNH, contato);
  • Cadastro e gestão de clientes (pessoa física e jurídica);
  • Cadastro e gestão de rotas (origem, destino, distância, pedágio);
  • Cadastro e gestão de cargas (peso, volume, tipo de mercadoria);
  • Programação, acompanhamento e atualização de status de viagens;
  • Cálculo automático de fretes com base em regras configuradas;
  • Emissão de relatórios gerenciais (viagens, fretes, utilização de frota);
  • Autenticação e controle de acesso por perfis de usuário;
  • Interface web responsiva (React);
  • Aplicativo mobile (React Native / Expo);
  • API REST com backend em Node.js;
  • Implantação em ambiente de nuvem.

4. Escopo Excluído

  • Integração com sistemas de rastreamento GPS em tempo real;
  • Módulo financeiro contábil (apenas cálculo de frete);
  • Integração com sistemas externos de NF-e / SEFAZ;
  • Aplicativo para clientes finais solicitarem fretes;
  • Módulo de manutenção de veículos.

6. Modelo de Dados (Entidades Principais)

O sistema contempla 8 entidades no modelo de dados:

#EntidadeDescrição
1VEICULOVeículos da frota com placa, modelo, marca, ano e capacidade de carga
2MOTORISTAMotoristas com CPF, CNH, contato e situação
3CLIENTEClientes pessoa física ou jurídica com CPF/CNPJ e contato
4ROTARotas com origem, destino, distância e valor de pedágio
5CARGACargas com descrição, peso, volume e tipo de mercadoria
6VIAGEMRegistro de viagens associando veículo, motorista, rota, carga e cliente
7FRETECálculo financeiro da viagem com valor base, pedágio e adicionais
8USUARIOUsuários do sistema com login, senha e perfil de acesso

8. Restrições e Premissas

  • O sistema não possui código legado — desenvolvido integralmente durante o semestre;
  • O banco de dados deve conter ao menos 8 entidades no modelo de dados;
  • O sistema deve ser acessível via web e mobile;
  • O sistema deve estar implantado em ambiente de nuvem;
  • A equipe utiliza Git para controle de versão durante todo o desenvolvimento.
